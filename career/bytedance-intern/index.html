SWE (intern) at ByteDance

Sep 2024 - Feb 2025

Worked on key management cloud system

Context

This was my final internship before graduating.

I joined the Security Engineering team, which worked across a wide range of systems including code signing, IAM, and key management.

My work

I worked on two projects:

1. Tag management for a key management product on Volcano Engine

  • built tagging framework from 0 to 1, while keeping it compliant with the surrounding cloud platform requirements
  • integrated tags into core product features such as access control, cost tracking, and resource management
  • shipped and used by external tech companies, with official documentation

2. Service reliability for an internal code-signing tool

  • took ownership of the reliability work for a new code-signing tool
  • proposed and implemented reliability practices including stress testing, automated end-to-end testing, and a multi-region deployment pipeline
  • drafted the product’s official SLI

My takeaways

One thing this internship made very obvious to me is that mature products (like Volcano Engine) come with rigorous requirements and compliance expectations. You are not just building something that works. You are building something that fits into a larger contract with users, operators, and the platform around it. This adds multiple dimensions onto the technical complexities.

I also got my first proper exposure to reliability engineering. It was different from the kind of work that I am drawn to, but very memorable still. I had vaguely understood it before, but this role made it feel concrete. It was one of the first times I really appreciated that good engineering is often about reducing the number of ways a system can fail, not just increasing the number of things it can do.
