Software researcher (intern) at A*STAR

May 2022 - Jul 2022

Worked on performance-critical C++ for homomorphic CNN inference

Context

This was my first internship in university, and it dropped me into a kind of problem that I still find very fun: heavy technical constraints, unusual theory, and performance work that actually matters.

I joined a research lab working on a homomorphic CNN for private image inference, which is a compact way of saying: we are trying to do ML without accessing user data for privacy reasons

The theoretical and design work was led by postdoctoral researchers. My role was to translate part of that design into efficient, reliable C++.

My work

I built a performance-critical data layer module in C++ that mapped tensors into ciphertext slots for downstream encrypted computation.

This sat directly on a hot path. The math was already fixed by the research design and was inherently expensive, so a lot of the performance came down to implementation quality: memory behavior, object lifetime, data movement.

The module was used in repeated research experiments where runtime translated pretty directly into iteration cost. Making it faster meant making the overall research loop cheaper and faster to run.

I liked this work because it felt like a clean example of engineering inside hard constraints where I had plenty of room to make the system meaningfully better.

My takeaways

This was one of the experiences that made me enjoy performance-oriented C++ work in the first place.

It taught me:

  • how much memory behavior and object lifetime decisions can affect runtime
  • how to structure low-level modules for both speed and maintainability
  • how to balance correctness, performance, and interface design in research-heavy systems
  • how to work with researchers by turning abstract or mathematically dense ideas into dependable code

Nowadays, I am diving back into C++.
