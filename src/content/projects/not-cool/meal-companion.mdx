## context

Meal Companion came from an everyday problem rather than a purely technical curiosity. That made it a useful reminder that software can be valuable simply by making a repeated routine less annoying.

## what I built

The application helps students track refrigerator items and plan meal preparation more effectively in a hostel environment.

## what mattered technically

The challenge here was mostly product framing and workflow clarity. Once the core user pain was clear, the technical work became about translating that routine into a simple and dependable interface.

## what I learned

This project reinforced that I enjoy software which sits close to real behavior. A modest idea can still be a good project if it teaches you how to make everyday systems feel cleaner.
