## context

I wanted a project that forced me to deal with the ordinary but important pieces of application building: state, user flow, payments, and the boundaries between frontend behavior and backend responsibility.

## what I built

My Shop is an e-commerce application that covers the basic path from browsing to checkout. The value of the project was not novelty, but the chance to build and reason through a complete product loop.

## what mattered technically

Commerce applications make integration details visible. Once payments and transactions enter the picture, seemingly simple user flows become coupled to data consistency and application state in more interesting ways.

## what I learned

This project helped me build stronger instincts around end-to-end product implementation and the importance of making backend behavior feel predictable from the user's point of view.
