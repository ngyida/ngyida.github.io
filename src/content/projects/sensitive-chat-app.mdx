## context

This project explored what happens when a familiar communication product is combined with emotional and sentiment-aware functionality. The core question was less about building a chat interface and more about what new interaction possibilities appear once language signals are part of the product.

## what I built

I built a messaging web application that integrated emotional recognition and sentiment analysis into the communication flow.

## what mattered technically

The interesting part was orchestration. The application needed to combine a normal real-time product surface with inference-driven behavior in a way that still felt coherent as a user experience.

## what I learned

I came away with a better sense of how ML-flavored features fit into product architecture: not as isolated models, but as behavior that has to be interpreted and surfaced carefully.
