## context

[Webnorm](https://sites.google.com/view/webnorm/home?authuser=0) is a newly proposed anomaly detection framework for web tampering. In plain terms, the goal was to identify when a website had been modified in suspicious ways, while keeping the detection process more structured and reliable than a simple bag of heuristics.

The research project was developed through a collaboration between researchers from [GovTech](https://www.tech.gov.sg/), [NUS](https://www.comp.nus.edu.sg/), and [SJTU](https://en.sjtu.edu.cn/).

## my contribution

Webnorm had promising initial results, but it also had some fairly obvious next questions around usability, robustness, and cost.

My contribution focused on three areas:

- improving the explainability of its results
- studying security concerns around the use of LLM components in the system
- reducing the cost of using the tool in practice

What made the project interesting to me was that those three concerns were not really separate. A system that is accurate but hard to interpret is difficult to trust. A system that is clever but easy to abuse is difficult to deploy. And a system that works but is too expensive to run is difficult to justify.

In short, the work was about making the tool feel more viable. The longer version of this project lives in my [dissertation](https://dl.comp.nus.edu.sg/handle/1900.100/18266), if you want the full write-up.
