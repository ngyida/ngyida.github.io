## Context

During my first-year undergraduate internship, I joined a research lab focused on machine learning and software security.
The team was building a high-performance homomorphic CNN for private image inference.

The theoretical and cryptographic design was led by postdoctoral researchers.
My role was to translate part of that design into efficient, reliable C++.

## My work


I built a highly performant data layer module in C++ that maps tensors into ciphertext slots for downstream encrypted computation.
This module sat in a performance-critical path. 
The math, fixed by the research design, was computationally expensive and implementation quality dictated runtime performance.

The module was deployed in large-scale research testing methodology that were costly, and the efficient runtime behaviour contributed to a lowered cost.


## My takeaways

Learned all about building a performance-oriented C++ system:
- How memory behavior and object lifetime decisions affect runtime
- How to structure low-level modules for both speed and maintainability
- How to balance correctness, performance, and interface design in research-grade systems
- How to collaborate effectively with researchers by turning complex theory into dependable code
