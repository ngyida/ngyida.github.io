## context

Skylab V2 was a good reminder that product work is often about shaping flow, not just adding features. The application had a real user journey behind it, which made API design and data decisions feel more consequential.

## what I worked on

I contributed across the stack, but the most meaningful work centered on:

- building APIs that supported new product capabilities
- refining database design to support those changes cleanly
- improving the user-matching workflow so the process became less manual and more coherent

## what mattered technically

Because the application had user-facing workflow complexity, backend changes could not be designed in isolation. Data models, endpoints, and interface expectations all needed to line up. That made this a useful chapter in learning how to think beyond a single layer.

## what I learned

This endeavor reinforced that I like full-stack work most when the backend remains the anchor. I enjoyed seeing how much better a product can feel when the underlying process is rethought instead of merely digitized.
